# Random Data Tools People

## Краткое Описание Сервиса

Random Data Tools People - web-сервис для загрузки случайных людей из внешнего API `https://api.randomdatatools.ru/`, сохранения данных в PostgreSQL и просмотра их через простые HTML-страницы.

На главной странице есть форма для загрузки нужного количества людей и таблица с пагинацией. Также доступны страницы конкретного пользователя и случайного пользователя.

## Стек

- Python 3.11.9
- FastAPI
- Jinja2
- SQLAlchemy
- Alembic
- PostgreSQL
- Pydantic Settings
- HTTPX
- Pytest
- Docker / Docker Compose

FastAPI выбран как легкий и быстрый web-фреймворк, который удобно использовать для простых HTML-страниц и при необходимости расширить до JSON API. PostgreSQL выбран как надежная реляционная БД для хранения большого количества записей и стабильной работы с пагинацией. SQLAlchemy отделяет бизнес-логику от конкретной БД, Alembic отвечает за версионирование схемы БД, а Jinja2 достаточен для простого HTML-интерфейса без усложнения frontend-части.

## Эндпоинты

- `GET /` - главная страница с формой загрузки и таблицей людей.
- `POST /load` - загрузка указанного количества людей из внешнего API.
- `GET /{user_id}` - страница конкретного пользователя по ID.
- `GET /random` - страница случайного пользователя.
- `GET /health` - проверка работоспособности сервиса.

## Архитектурный Подход

Проект разделен на простые слои:

```text
app/
  api/            зависимости FastAPI и роуты
  core/           настройки, сборка приложения, startup-события
  db/             подключение к БД и базовый класс моделей
  models/         SQLAlchemy-модели
  repositories/   функции доступа к данным
  services/       бизнес-логика и клиент внешнего API
templates/        HTML-шаблоны
static/           CSS
tests/            тесты
alembic/          миграции БД
```

Сборка приложения находится в `app/core/app_factory.py`, стартовая загрузка данных - в `app/core/events.py`. Роуты не работают с SQL-запросами напрямую, а обращаются к сервисному слою. Неиспользуемый слой `schemas` удален, так как сервис сейчас отдает HTML-страницы, а не JSON API.

## Используемые Паттерны

- **Factory Method**: приложение создается через `create_app()`. Это упрощает запуск и позволяет в тестах создать приложение с тестовой БД и замоканным API-клиентом.
- **Repository**: модуль `repositories/people.py` скрывает SQL-запросы и операции с таблицей `people`.
- **Service Layer**: модуль `services/people.py` содержит бизнес-логику загрузки, нормализации, пагинации и выбора случайного пользователя.
- **Dependency Injection**: FastAPI-зависимость `get_db()` выдает сессию БД, а `get_api_client()` позволяет подменять внешний API в тестах.
- **Mock / Fake Object**: в тестах внешний API заменен на `FakeApiClient`, поэтому тесты не зависят от интернета и стороннего сервиса.
- **Migration**: Alembic хранит историю изменений схемы БД и позволяет воспроизводимо создавать таблицы.

## Запуск

Создайте `.env` из примера:

```bash
copy .env.example .env
```

### Запуск Через Docker

```bash
docker compose up -d --build
```

Сервис будет доступен по адресу:

```text
http://127.0.0.1:8000
```

## Тесты

Запуск тестов:

```bash
pytest
```

Во время тестов внешний API не вызывается: вместо него используется fake-клиент с заранее известными данными. База данных для тестов создается в памяти через SQLite.

```text

```
